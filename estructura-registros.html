Descripción del proyecto

Contexto del e-commerce y problema que resuelve

MundialMarket es un marketplace dedicado a la compraventa e intercambio de objetos de colección relacionados con los Mundiales de Fútbol. Los usuarios pueden publicar y adquirir figuritas repetidas, camisetas de ediciones históricas, pins oficiales, álbumes y otros artículos coleccionables.

Para modelar la información de cada artículo publicado en el sistema, se diseñó el registro ARTICULO_MUNDIALISTA, que permite almacenar de forma estructurada y heterogénea todos los datos relevantes de un artículo, identificarlo de manera única y relacionarlo con el vendedor correspondiente.

🔑
Justificación de la clave primaria: id_articulo — N(8)
Se eligió id_articulo como clave primaria porque es un número entero autogenerado que garantiza unicidad total para cada registro en la secuencia.

Se descartó el campo nombre porque dos artículos pueden tener el mismo nombre (por ejemplo, dos figuritas "Maradona 1986" de distintos vendedores). Se descartó id_vendedor porque un mismo vendedor puede publicar múltiples artículos.

El campo id_articulo es una clave primaria simple (campo contenido de tipo N(8)), lo que lo hace eficiente para la búsqueda y ordenamiento en la secuencia de registros. Por su parte, id_vendedor actúa como clave foránea, vinculando este registro con el archivo VENDEDORES y permitiendo relaciones entre archivos.
Definición literaria del registro

Pseudocódigo al estilo de la cátedra de Algoritmos y Estructuras de Datos

ACCION MundialMarket ES;
AMBIENTE

ARTICULO_MUNDIALISTA = Registro
    id_articulo        : Entero;              {CLAVE PRIMARIA — única por artículo}
    nombre              : AN(100);
    categoria           : ("Figurita","Camiseta","Pin","Album","Otro");
    mundial             : AN(30);
    pais_equipo         : A(30);
    fecha_publicacion = Registro              {CAMPO CONTINENTE}
        dia  : Entero;
        mes  : Entero;
        anio : Entero;
    Fin Registro;
    precio              : Real;
    modo_operacion      : ("Venta","Intercambio","Ambos");
    estado_conservacion : ("Mint","Muy bueno","Bueno","Regular");
    es_raro             : Logico;
    id_vendedor         : Entero;              {CLAVE FORÁNEA → registro VENDEDOR}
Fin Registro;

ALGORITMO
    {Ejemplo de uso del selector de campo:}
    Reg.id_articulo := 10000001;
    Escribir(Reg.nombre);
    Escribir(Reg.fecha_publicacion.dia, Reg.fecha_publicacion.mes, Reg.fecha_publicacion.anio);
FIN.
Clasificación de campos y claves

Resumen de los conceptos del Tema 7 aplicados en el diseño

Campos contenidos

Campos simples que almacenan un único dato elemental. En este registro: id_articulo, nombre, precio, es_raro, entre otros.

Simples

Campo continente

fecha_publicacion es un campo que contiene a otros campos más simples: dia, mes y anio, cada uno de tipo N.

Estructurado

Clave primaria

id_articulo identifica de forma única e irrepetible cada artículo publicado en el sistema.

Única · Simple

Clave foránea

id_vendedor es una clave secundaria que relaciona el registro ARTICULO con el registro VENDEDOR de otro archivo.

Relación entre archivos

Heterogeneidad de datos

El registro combina tipos Entero, Real, AN, A, Lógico y ENUM en un mismo tipo de dato estructurado.

Propiedad del registro

Selector de campo

Se usa el operador punto ( . ) para acceder a los campos: Reg.nombre, Reg.fecha_publicacion.dia.
